# Memo Template Skill

## 📋 嚴格規矩

### **規矩1：不能多次創建**
- 只有一個模板 JS 文件
- 不能無限擴展

### **規矩2：必須跟 skill LLM 搭建**
- 使用 skill 系統管理
- 避免 path 管理錯誤

### **規矩3：不要自行編寫**
- 必須按照規則執行
- 不能隨意添加功能

### **規矩4：不要估我需要點做**
- 不能猜測用戶需求
- 必須等待明確指令

### **規矩5：只執行我的命令**
- 嚴格按照用戶指令執行
- 不能主動添加功能

## 🚫 禁止行為

### **絕對禁止**：
- ❌ 創建多個模板文件
- ❌ 手動管理路徑
- ❌ 自行編寫額外功能
- ❌ 猜測用戶需求
- ❌ 主動添加功能

### **必須等待**：
- ✅ 用戶明確的模板操作指令
- ✅ 具體的功能需求說明
- ✅ 明確的執行步驟要求

## 🔧 唯一模板功能

**文件位置**：`tools/memo/唯一模板.js`

### **嚴格限制的功能**：
1. **讀取模板結構** - 僅讀取，不分析
2. **生成 Excel 文件** - 僅生成，不修改

### **禁止的功能**：
- ❌ AI 分析（除非明確要求）
- ❌ 對話系統（除非明確要求）
- ❌ 自動優化（除非明確要求）

## 📝 使用流程

### **正確使用**：
```javascript
// 1. 用戶明確要求創建模板
const template = new MemoTemplate();

// 2. 用戶明確要求確認模板
template.confirmTemplate();

// 3. 用戶明確要求生成文件
await template.generateExcelFile(userData);
```

### **錯誤使用**：
```javascript
// ❌ 禁止：主動添加 AI 分析
// ❌ 禁止：猜測用戶需要對話系統
// ❌ 禁止：自行編寫額外功能
```

## 🎯 觸發條件

**只有當用戶明確要求以下操作時才觸發**：
- "創建 Site Memo 模板"
- "生成 Site Memo Excel 文件"
- "讀取模板結構"

**不觸發的情況**：
- 用戶沒有明確提及模板操作
- 用戶只是詢問相關信息
- 用戶需要其他類型的功能
